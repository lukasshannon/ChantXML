Minimal MusicXML Extension Plan for Gregorian Chant

This plan defines how ChantXML should extend or profile MusicXML with the smallest practical set of additions needed to preserve Gregorian chant semantics and enable reliable conversion with GABC and TeX workflows.

Design principles

  • Minimality: Use existing MusicXML elements whenever possible.
  • Interoperability: Keep documents readable by generic MusicXML tooling.
  • Round-trip fidelity: Preserve enough detail for gabc → chantxml → gabc.
  • Explicit metadata: Store chant-only details in namespaced extension attributes/elements.
  • Traceability: Keep source token hints to improve deterministic back-conversion.

Proposed extension strategy

ChantXML should begin as a profile of MusicXML 4.0 plus a small namespaced extension set (provisionally xmlns:chant="https://chantxml.org/ns/1"). The profile defines constraints, while the namespace carries chant-specific data not safely representable in base MusicXML.

Where extensions are expected

MusicXML area Use in ChantXML Extension need
<note> and pitch info Represent chant pitch events and neume grouping anchors Neume/group IDs, liquescent flags, ornamental marks
<notations> Attach neume/rhythmic semantics to notes Chant-specific notation vocabulary
<lyric> Text underlay and syllable alignment Extended underlay behavior and melisma hints
Score-level metadata Mode, office/mass context, source references Chant metadata block and source provenance

Feature mapping plan (GABC ↔ ChantXML)

Feature Primary MusicXML representation Chant extension payload Round-trip notes
Neume groupings Sequential notes with grouping conventions chant:neume-id, chant:neume-type Preserve original grouping boundaries from GABC tokens
Liquescent forms Note-level notation markers chant:liquescent="true|type" Must retain subtype for deterministic GABC regeneration
Episema / ictus-like marks Notation attachments chant:rhythmic-mark vocabulary Allow multiple marks per neume when needed
Text underlay behavior <lyric>, syllabic structure chant:underlay-rule Store explicit hints where base mapping is ambiguous
Source token provenance n/a chant:source-token Optional but recommended for lossless re-export

Normative profile deliverables

  1. ChantXML Profile Spec: required/forbidden MusicXML constructs for chant files.
  2. Extension Namespace Spec: definitions for all chant:* fields.
  3. Mapping Tables: canonical GABC token mappings and edge-case behavior.
  4. Conformance Levels: importer/exporter support levels and compatibility targets.
  5. Examples Corpus: curated chant fixtures with expected round-trip output.

Milestone plan

M1 — Core profile

  • Freeze initial namespace and required metadata fields.
  • Document minimum note/lyric mapping semantics.
  • Publish 10 representative chant examples.

M2 — Round-trip stability

  • Implement and validate gabc → chantxml → gabc converter path on fixture corpus.
  • Track and classify lossy cases; add explicit metadata where needed.

M3 — TeX publishing bridge

  • Implement chantxml → tex aligned with GregorioTeX assumptions.
  • Document any unsupported notational constructs and fallback behavior.

Illustrative extension snippet (non-normative)

<note chant:neume-id="n12" chant:neume-type="clivis" chant:source-token="(gh)">
  <pitch>...</pitch>
  <notations>
    <chant:rhythmic-mark type="episema-horizontal" />
  </notations>
  <lyric chant:underlay-rule="extended-melisma">...</lyric>
</note>

Exact element and attribute names will be finalized in the ChantXML profile draft.
