ChantXML

A minimal MusicXML extension profile for Gregorian chant, with practical conversion pathways to and from GABC and TeX.

Project priorities

  • Minimally extend MusicXML rather than creating a new format.
  • Support round-trip conversion with existing GABC chant corpora.
  • Enable TeX-based chant publishing workflows (GregorioTeX compatible).
  • Target practical interchange with mainstream notation tools, including Dorico and Sibelius.
  • Reuse existing open-source projects with full attribution.

Roadmap (documentation phase)

  1. Draft ChantXML profile and mapping rules.
  2. Document use of MusicXML <staff-lines>4</staff-lines> for chant staves.
  3. Evaluate MusicXML <solo/> instrument metadata for solo-voice chant examples.
  4. Support movable chant clefs with MusicXML <sign>C</sign> / <sign>F</sign> and flexible <line> values.
  5. Publish canonical conversion examples.
  6. Define converter MVP command workflows.
  7. Host browser-based docs and tooling in this site.

Planned conversion flow

gabc  -> chantxml
chantxml -> gabc
chantxml -> tex

Round-trip fidelity is a first-class requirement. Any intentional loss of information should be explicitly documented.

Reference ecosystem

The project is designed to interoperate with the existing chant ecosystem:

  • MusicXML 4.0
  • MusicXML <notehead> reference (SMuFL attribute support)
  • SMuFL medieval and renaissance clefs (important for chant clef compatibility and usable in MusicXML via SMuFL-linked glyph metadata).
  • SMuFL medieval and renaissance plainchant glyph table
  • Gregorio Project
  • GABC format
  • GregorioTeX
  • TeXShop (the intended LaTeX editor/program target for ChantXML-to-TeX output workflows).
  • GregoBase corpus

Viewer chant conversion behavior

The viewer includes a Convert to Gregorian Chant button that updates the MusicXML in-place using minimal MusicXML features.

Input example (normal notation)

<attributes>
  <clef>
    <sign>G</sign>
    <line>2</line>
  </clef>
</attributes>

Output example (chant-oriented notation)

<attributes>
  <staff-details><staff-lines>4</staff-lines></staff-details>
  <clef>
    <sign>C</sign>
    <line>3</line>
  </clef>
</attributes>
<notehead smufl="chantPunctum" font-family="Bravura">other</notehead>

Assumptions and edge cases

  • The converter updates only the first measure's <attributes> block.
  • Every existing <notehead> is normalized to other with medieval plainchant SMuFL metadata smufl="chantPunctum" and a font-family="Bravura" hint; for pitched notes missing a notehead, one is inserted.
  • The viewer loads Bravura and Bravura Text before rendering and sets Bravura as the default notation font so SMuFL chant glyphs are available when the score is drawn.
  • Font loading is local-first: the viewer checks docs/assets/fonts/bravura/ before CDN fallback URLs, supporting offline/self-hosted deployments where external font URLs may be blocked.
  • Bundled font attribution and license metadata are tracked in docs/assets/fonts/bravura/README.md and docs/assets/fonts/bravura/LICENSE.txt.
  • Renderer behavior can still vary across MusicXML engines for smufl notehead overrides, but explicit font loading plus font hinting improves compatibility in common web setups.

Round-trip expectations

This is a one-way viewer transform for exploration. A full gabc -> chantxml -> gabc round-trip converter is still planned and may treat notehead glyph metadata as optional display-level data.
